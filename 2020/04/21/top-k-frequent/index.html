top k frequent

2020-04-21 计算机 数据结构 0 评论

寻找数据流中出现最频繁的k个元素（find top k frequent items in a data stream）,这个问题也叫heavy hitter。

方案一：HashMap + Heap

用一个HashMap记录每个元素出现的次数，用一个小根堆统计目前出现最频繁的k个元素。具体操作如下：

  1. 每次来一个元素，如果在HashMap里已经存在，则把对应的计数器加1；如果不存在，则把元素插入HashMap中，并把相应的计数器设置为1.
  2. 在堆中查找该元素，如果存在，则把堆里的计数器也加1，并调整堆；如果没有找到，则把这个元素的次数和堆顶元素进行比较，如果大于堆顶元素出现的次数，则用此元素替换堆顶元素，并调整堆。
  3. 空间复杂度O(n)。HashMap需要存放下所有元素，需要O(n)的空间，堆需要存放k个元素，需要O(k)的空间，跟O(n)相比可以忽略不急，总的时间复杂度是O(n)
  4. 间复杂度O(n)。每次来一个新元素，需要在HashMap里查找一下，需要O(1)的时间；然后要在堆里查找一下，O(k)的时间，有可能需要调堆，又需要O(logk)的时间，总的时间复杂度是O(n(k+logk))，k是常量，所以可以看做是O(n)。

如果元素数量巨大，单机内存存不下，怎么办？

方案二： 多机 HashMap + Heap

  1. 可以把数据进行分片，假设有8台机器，第1台机器只处理hash(elem)%8==0的元素，第2台机器只处理hash(elem)%8==1的元素，以此类推。

  2. 每台机器都有一个HashMap和一个 Heap, 各自独立计算出 top k 的元素。

  3. 把每台机器的Heap，通过网络汇总到一台机器上，将多个Heap合并成一个Heap，就可以计算出总的 top k 个元素了。
