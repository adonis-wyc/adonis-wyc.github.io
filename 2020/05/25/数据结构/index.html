    • 数据结构

    2020-05-25 计算机 数据结构 0 评论
    • 在内存中，数组是一块连续的区域
    • 数组需要预留空间，在使用前先申请占内存的大小
    • 插入和删除数据的效率很低。插入数据时，这个位置后面的数据在内存中都要向后移；删除数据时，这个位置后面的数据都要向前移
    • 随机读取效率很高。因为内存是连续的，知道每一个数据的内存地址
    • 不利于扩展，数组定义的空间不够时需要重新定义数组

    链表的特点

    • 在内存中不需要连续
    • 每一个数据都保存了下一个数据的内存地址，通过这个地址找到下一个数据。
    • 增加和删除数据只需要通知相邻数据，比较容易
    • 查找数据时效率低，因为不具有随机访问性
    • 不指定大小，容易扩展

    数组的优点

    • 可以随机查找
    • 查找速度快

    数组的缺点

    • 插入和删除效率低
    • 内存空间要求高，必须有足够的内存空间，并且可能浪费内存
    • 数组大小固定，不能动态拓展

      桶排序之计数排序：

      • 得到无序数组的取值范围
      • 根据取值范围创建对应数量的桶
      • 遍历数组，把每个元素放到对应的“桶”中
      • 按照顺序遍历桶中的每个元素，依次放到数组中，即可完成数组的排序

      桶排序之基数排序（Radix sort）：

      • 将所有待比较数值统一为同样的数位长度，数位较短的数前面补零
      • 从最低位开始，依次进行一次排序
      • 从最低位一直到最高位排序完成以后，数列就变成了一个有序序列

    二叉树、B树、B+树

    1. 平衡二叉树

    非叶子节点最多拥有两个子节点；

    非叶子节点值大于左边子节点，小于右边子节点；

    每个节点左右子树高度差不超过1；

    没有值相等重复的节点。

    ![image-20200414124358658](/Users/wangyuchen/Library/Application Support/typora-user-images/image-20200414124358658.png)

    2.B树

    描述B树的时候用字母m表示阶数（M阶代表一个树节点最多有多少个查找路径），根结点关键字数量范围：1<= k <=m - 1,非根结点关键字数量范围：m/2 <= k <= m - 1;

    每个节点中的关键字按从小到大的顺序排列，每个关键字的左子树中的所有关键字都小于它，右子树中的所有关键字都大于它；

    所有叶子节点位于同一层；

    每个节点都存有索引和数据，也就是对应的key和value。

    ![image-20200414124434086](/Users/wangyuchen/Library/Application Support/typora-user-images/image-20200414124434086.png)

    B树的查询流程

    从根结点以此向下进行根节点比较，比较规则遵循二分法。

    B树插入节点流程

    （1）节点拆分规则：关键字数 > 4就要进行节点拆分，用节点的中间key将这个节点分为左右两个部分，中间节点放到父亲节点中即可。

    （2）排序规则：满足节点本身比左边节点大，比右边节点小的规则排序。

    B树的删除操作

    （1）节点合并规则，节点字数必须要大于等于（M／2），小于（M／2）时先从子节点取，子节点没有符合条件的就向父节点取，取中间值往父节点放。

    （2）排序规则：满足节点本身比左边节点大，比右边节点小的排序规则。

    3. B+树

    非叶子节点的分支数等于关键字数。根结点关键字个数为1<= n <=m,非根节点关键字个数为ceil(m/2) <= n <=m.

    B+树有两种类型的节点：内部节点（也称索引节点或非叶子结点）和叶子节点。内部节点不存储数据，只存储索引，数据都存储在叶子节点。

    内部节点中的key都按照从小到大的顺序排列，对于内部节点的一个key，左子树中所有key都小于它，右子树中的key都大于等于它。叶子节点中的key也是按从小到大的顺序排列。

    每个叶子节点都存有相邻叶子节点的指针，叶子节点本身按照key的大小从小到大的排列连接。

    父节点（注意是父节点不是跟节点）存有右孩子第一个元素的索引。
