Java设计模式

2020-07-03 计算机 Java设计模式 0 评论

Java设计模式

创建型模式

单例模式:

JDK：JDK中的Runtime运用了单例模式，通过调用getRuntime()获得Runtime实例，可以调用Runtime()对象的方法去查看Java虚拟机的状态以及控制虚拟机的行为。

Spring：Spring中的bean默认采用单例模式。

饿汉式（线程安全）：类加载时就创建单例对象。

懒汉式（线程不安全）：在单例对象第一次使用时才创建。(非线程安全版本)

工厂方法模式

定义一个用于创建产品的接口，由子类决定生产什么产品。

例如：

  1. Object类里的toString()方法，Java的任何类，都可以继承或者覆写toString()方法，Object相当于抽象工厂类，各个Java的类相当于具体工厂类。

  2. Class类中的newInstance()方法。

  3. Collection中的iterator()方法是一个工厂方法，对于iterator()方法来说Collection就是一个根抽象工厂，往下有ArrayList等具体工厂，Iterator是根抽象产品，往下有ArrayListIterator等作为具体产品。

###抽象工厂模式

定义一个用于创建产品族的接口，每个子类可以生产一系列相关的产品。

例如：

  1. java.sql.Connection代表了抽象工厂类，里面有创建各个产品类的函数，具体的产品实现类，具体的Connection工厂封装在各个数据库驱动包中，通过Connection我们就可以创建Statement, Clob等同一产品族的对象。

构造者模式

定义：使用多个简单的对象一步一步构建成一个复杂的对象。

例如：java.lang.StringBuilder#append()

原型模式

定义：用于创建重复的对象，同时又能保证性能。

例如：java.lang.Object#clone()

结构型模式

代理模式

客户端通过代理间接的访问被代理的对象，从而限制，增强，或修改该对象的一些特性。

适配器模式

将一个类的接口转换成客户希望的另一个接口，使得原本由于接口不兼容而不能一起工作的那些类能一起工作。

装饰模式

动态的给对象增加一些额外的职责。

行为型模式

策略模式

定义了一系列算法，并将每个算法封装起来，使他们可以相互替换，且算法的改变不会影响使用算法的客户。

###观察者模式

多个对象间存在一对多关系，当一个对象发生改变时，把这种改变通知给其他多个对象，从而影响其他对象的行为。

例如：

  1. java.util.Observer/jave.util.Oberservable

责任链模式

定义：为请求创建一个接收者对象的链，每个接受者都包含对另一个接收者的引用。如果一个对象不能处理该请求，那么它会把相同的请求传给下一个接收者，以此类推。

例如：java.util.logging.logger#log()

Spring中运用到了哪些设计模式

###工厂方法模式

Spring使用工厂模式通过BeanFactory或ApplicationContext创建bean对象。

单例模式

Spring中bean的默认作用域就是singleton(单例)的。Spring通过ConcurrentHashMap实现单例注册表的特殊方式实现单例模式。

代理模式

AOP中运用了代理模式。

模版方法模式

定义：模版方法模式是一种行为设计模式，它定义了一个操作中的算法的骨架，而将一些步骤延迟到子类中。

Spring中的jdbcTemplate，hibernateTemplate等以Template结尾的对数据库操作的类，就用到了模版模式

观察者模式

Spring事件驱动模型运用了观察者模式，可以解耦代码。

流程：

  1. 定义一个事件，继承自ApplicationEvent,并且写相应的构造函数。
  2. 定义一个事件监听者，实现ApplicationListener接口，并重写onApplicationEvent()方法。
  3. 使用事件发布者发布消息，可以通过ApplicationEventPublisher的publishEvent()方法发布消息。

包装器设计模式

适配器模式

Spring AOP的通知使用到了适配器模式，Advice(通知)的类型有：BeforeAdvice, AfterReturningAdvice，ThrowsAdvice。每个通知都有对应的拦截器，MethodBeforeAdviceInterceptor, AfterReturningAdviceInterceptor, ThrowsAdviceInterceptor。Spring需要使用适配器模式对Advice进行转换，将Advice封装成对应的拦截器类型，返回给容器。

Spring MVC中的适配器模式。

DispatcherServlet根据请求信息调用HandlerMapping，解析请求对应的Handler。之后由HandlerAdapter适配器处理，得到具体的Controller。

装饰者模式

Spring中配置DataSource的时候，DataSource可能是不同的数据库和数据源。装饰者模式可以根据客户需求在少有修改原有类的代码下动态切换不同的数据源。

Java内部类

好处：变相的实现多重继承，并且便于维护。

分类

静态内部类：内部类不可以访问外部类的非静态变量和方法，要想访问的话要通过外部类的实例。

非静态内部类：内部类可以直接访问外部类的成员变量和方法。

本地内部类：本地内部类声明在一个块中，是函数中声明的一个类，作用域只在函数内，外部没办法创建实例。

匿名类：没有名字的内部类，用来重写一个类的方法，或者实现一个接口。

Object有哪些方法

  1. 反射相关：getClass获取类对象。

  2. 线程间通信：wait（） ：释放对象锁，并将此线程放入等待队列。

    ​ notify（）／notifyAll（）：通知等待此对象的等待队列的线程进入同步队列，准备获取对象锁。

  3. 用于垃圾清理： finalize（）

  4. hashCode（）和 equals（）：hashCode用于返回对象的哈希值。equals（）用于判断在对上的对象是否相同。

  5. clone（）和toString（）：clone（）返回此对象在堆上创建的副本。toString（）返回对象的字符串表示。
