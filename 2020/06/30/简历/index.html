课程管理系统

数据库中的结构

学生信息表： ** **学号，姓名，年龄，年级。

课程信息表： 课程号，课程名称。

学生课程表：学号，课程号。

  1. 前端使用angular 6框架，可以使用ngif和ngfor，生成html，CSS，ts文件，当用户在使用界面点击按钮时，angular调用ts文件中的方法，调用一个共享的ts文件， 向指定的url发送http请求。

房屋推荐系统

项目背景：设计一个类似airbnb的找房软件，租房的人可以输入自己的筛选条件找到符合条件的房屋，房东可以得到自己房屋的预测价格。

挑战：学习pandas和flask，学习mapboxGL API。

我在项目中的作用：数据清理，编写datahandle函数，编写每个功能的逻辑

创新点：使用mapboxGL API将位置显示在地图上。

功能1:将所有房屋都显示在地图上，使用者将鼠标放在地图上的点可以得到房屋的具体信息。

功能2:用户输入自己的筛选条件，提交表单数据，得到符合条件的房屋推荐并显示在地图上。

功能3:房东输入房屋信息得到房子的预测价格。

输入是三个csv文件（大小都在50MB左右）：

calendar.csv（特定日子房屋的价格），

listings.csv (房屋的详细信息，例如位置，房屋类型等),

reviews（房屋的评价信息）.csv

flask中使用@app.route()写API，return render_template()来返回页面。

flask中使用request.form.get()得到表单数据。

利用pandas先对数据做清洗，选择需要的列，并把数据更改成易于处理的类型。编写datahandle函数，输入筛选条件，然后用pandas进行判断，返回满足条件的数据。

使用flask框架开发网页，利用pandas工具包在后端处理数据。

营地分享网站

Node.js：运行javascript的服务器。

框架使用Express，

数据库采用mongodb，与数据库交互时采用mongoose，将数据库中的实体编写成类，调用预先写好的方法

数据库划分为三个实体，评论：用户，内容；营地：名字，图片，描述，评论，作者。 使用者：用户名，密码。

把每一个实体都定义成javascript中的一个model。

使用中间件，其中调用passport-local工具包的验证方法判断当前用户是否有权利修改营地信息和评论信息。

页面显示使用的是EJS模版引擎，让node.js渲染画面。

  1. nosql是基于键值对不需要经过sql层的解析，性能非常高。

  2. 数据之间没有耦合性，容易水平扩展。

sql：

  1. 可以做复杂查询。
  2. 事务支持，安全性高。

机票预订系统

项目背景：课程大作业。

挑战：第一次将后端与数据库进行连接，并且要自己设计数据库中的表格。

我在项目中的作用：设计数据库的结构，添加数据，将后端与数据库进行连接，对相应的功能编写sql语句，用preparestatement进行封装。编写各个功能的时候进行相应的逻辑判断并调用封装好的sql语句。隔离级别采用可重复读。

创新点：将数据库部署到云端服务Azure上，提高数据存储能力。

数据库中的表：

  1. 航班信息表(航班号，出发日期，到达日期，始发地，目的地，航空公司，载客量，价格)
  2. 订单表(订单号，是否支付，花费，航班号，用户名)
  3. 用户信息（用户名，密码，余额）。

实现用户注册，登陆，查询航班，预定，付款等功能。

前端显示是用java从命令行读取指令转化成相应的查询功能并进行调用。并编写测试用例，利用shell文件对代码进行编译然后在Junit框架上运行。

电影推荐系统

项目背景：通过netflix数据集建立一个电影推荐系统。

挑战：学习hadoop生态中的工具，例如mapreduce，HDFS，还有使用docker搭建mapreduce。

我在项目中的作用：确定使用何种推荐算法，对计算过程进行建立两个矩阵相乘的数学模型。

创新点：利用docker容器搭建mapreduce。

docker容器：在本机搭建hadoop需要配置依赖，配置参数，和电脑兼容，封装在docker容器中，可以被很多人方便的使用。

docker VS VM：VM有自己的OS，container用主机的OS，更轻量级。

HDFS：

如何存储：将数据进行拆分和复制。

整体架构：master接受写文件请求，告诉客户端将拆分后的文件以及对应的副本存储到对应的slave机器上。

写操作：先建立slave node之间的data queue，当所有slave node准备就绪时，将内存中的数据写入slave node。

选择item CF，基于电影的相似性作出推荐，而不是用user CF，因为用户的数量远大于电影的数量。

如何建立电影之间的关系：采用用户的打分记录，建立同现矩阵，表示有多少用户同时给两部电影打过分。同现矩阵要进行归一化处理，将两个电影之间关系的绝对值转化为相对值，也就是在计算电影1与电影2关系时，应该计算电影1与电影2的关系值比上与其他所有电影的关系值的和。

如何对用户做个性化推荐：根据用户的打分记录，建立评价矩阵，表示每部电影收到不同用户的评分。

协同矩阵和评价矩阵做乘法得到用户对每个电影可能的打分。

map reduce中间进行shuffle过程，对map任务后的数据进行分割，分区，和排序。reduce shuffle将同一分区的数据进行合并。

进行map reduce

  1. Mapper：Input：key: null value :user_id, movie_id, rating Output: key: user Value: movie : rating

    Reducer: Input: key: user Value: movie : rating Output: key: user Value: movie : rating

  2. Mapper：Input：key: user_id, value: movie : rating Output: key: movieid : movieid value: 1

    Reducer: Input: key: movieid: movieid value 1 Output: key: movieid: movieid Value: sum

  3. Mapper: Input: movieA : movieB \t relation Ouput: key : movieA OutputValue: movieB = relation

    Reducer: Input: key : movieA Value: movieB = relation Output: key: movieB value: movieA = normalizedRelation

  4. Mapper1: Input: movieB\moviedA = normalizedRelation Output: key: movieB value: movieA = relation

    Mapper2: Input: user, movie, rating Output: key: movie. value: user : rating

    Reducer: Input: key : movieB value: movieA = relation … user : rating. Output: key : user : movieA value: subrating

  5. Mapper: Input: user1 : movie2 \t subrating Output : key: user1 : movie2 value: subrating

    Reducer: Input: key: user1 : movie2 value: subrating Output: key: user1 : movie2 value: sum.

极象棋机器人对战平台

把游戏看成一个树状图，往后推七步的所有可能性列出来，并根据static evaluation算法给出每种可能性的分数。我希望最大化这个分数，而我的对手需要最小化这个分数。利用minmax算法，从结果看起，比如第八步是我的对手下棋，他要最小化分数，于是根据结果反推做出选择，根据对手选择的结果我们可以反推出自己的选择，依此类推可以发现第一步的最优选择。在过程中可以根据选择最大或最小的性质进行alpha-beta的剪枝，减小运算量。
